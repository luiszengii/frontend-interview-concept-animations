# 更新记录

本项目遵循“面向用户的变化优先”记录版本。

## 1.5.2 - 2026-07-17

- 大屏题库从等高双列升级为自然高度瀑布流，短卡片下方不再因为相邻长卡片留下空白。
- 瀑布流保留题目 DOM、键盘和屏幕阅读器顺序；卡片展开、筛选或跨越 1280px 断点时自动重新排布，平板与手机继续单列展示。

## 1.5.1 - 2026-07-17

- 题库在 1280px 及以上的大屏视口使用双列卡片布局，并针对半宽卡片收紧标题、标签、信息条与操作区尺寸。
- 平板与手机继续保持单列布局，避免牺牲阅读和触控体验。

## 1.5.0 - 2026-07-15

- 为全部 56 道题增加至少两段可直接学习和复述的正式回答，不再只有关键词骨架。
- 题库卡片按参考稿重构为题型、公司、骨架、操作与来源五层信息结构，并适配手机端单列浏览。
- 题库卡片与独立题目页使用默认收起的正式回答区，兼顾手机端浏览密度。
- 内容校验新增 `fullAnswer` 完整性约束，防止后续新增题目退化为只有回答提纲。

## 1.4.0 - 2026-07-14

### 面试复盘与题链

- 录入字节跳动（OD）前端面试复盘，8 道真实问题均关联可追溯的公司出现记录。
- 题库从 18 道扩展到 56 道，围绕 AI 多轮对话、React、多租户、Next.js、请求层、RBAC、Skill、CI/CD 和性能建立继续深挖题链。
- 题目详情页新增“继续深挖”，同一主题可沿相关问题逐层训练。

### 新动画

- 新增 AI 多轮对话运行时：消息 ID、乐观更新、cursor 历史、虚拟窗口、prepend 与动态高度滚动锚点。
- 新增 AI 结构化消息协议：字节流、SSE 事件边界、typed parts、卡片增量更新、未知版本安全降级。
- 新增二进制文件流恢复：stall 检测、checkpoint、Range/If-Range、206、ETag 失配与完整性校验。

### 页面联动

- 有动画的文字题卡显示明确按钮，并在新页面打开对应独立 scene URL。
- 内容校验新增复盘与公司次数双向一致、相关题 ID 和动画 scene ID 检查。

## 1.3.0 - 2026-07-14

### 重做

- 将 AI Token 流、上下文压缩、ReAct/Skill、多租户、Next.js 请求链、Axios/SSE 管道、RBAC 七组动画从通用流程模板改为概念专属舞台。
- 新场景分别使用三层时钟、上下文工作台、推理环路、单镜像投影、鉴权分支、传输分流与三道安全门，不再只替换卡片文案。

### 修正

- 修正 `let` 暂时性死区示例在未捕获异常后继续执行的错误叙事。
- 修正循环引用 JSON 序列化的异常类型，并补充 `structuredClone` 的适用边界。
- 修正 React index key 的表述：问题是组件状态与业务记录错配，不是必然大面积销毁重建。
- 按当前 Next.js 文档将 Middleware 标注为 Proxy 的旧称，并保留面试常用术语。

### 来源与维护

- 为本轮修正与重做场景补充 MDN、React、Next.js、OWASP、OpenAI 等官方来源。
- 场景维护指南禁止共享流程模板，并要求概念专属空间隐喻、持续对象和失败路径。

## 1.2.0 - 2026-07-14

### 新增

- 增加 design-interview-animations Skill，以五组高质量动画为金标准。
- 增加动画质量评分表，禁止通用卡片模板替代概念设计。
- 每道题增加独立详情 URL，每个动画场景增加稳定 scene URL。
- 题目详情页展示回答骨架、公司次数、来源和关联动画。

## 1.1.0 - 2026-07-14

### 新增

- 增加题库与复盘独立入口，支持搜索、方向、题型与来源状态筛选。
- 使用结构化数据记录题目、技术出处、公司出现次数和面试复盘。
- 每道题可关联概念动画，并展示可查询来源。
- 增加 interview-recap-curator Skill，用于把面试复盘沉淀为题库记录。
- 增加 GitHub Pages 自动部署工作流。

## 1.0.0 - 2026-07-14
